This is how I originally went about building the User Manager. I thought it made sense to pass the selected user as a prop to UserForm rather than the selectedUserId. As I was going down this rabit hole, it led me to break a basic rule of Vue, which is never mutate a prop

We could try to avoid breaking this rule by making a copy of of the prop, and storing the property as a data member. this will require use to rename the prop to 'initialUser' instead of 'user' (we'll also have to update the template in the root component). This is actually a solution that is recommended in Vue. For more details, checkout this link. But note that it defaults to showing the code samples using the 'composition api', and we are using the 'options api'. So you can press a toggle switch at the top left of the page to show the code samples using the options api.

Let's try to follow the suggestion and see if we can get the app to work.
